# Admin Dashboard Documentation

## Overview

The Admin Dashboard is a comprehensive management interface for system administrators to oversee users, projects, and invite tokens. It provides real-time statistics, user management capabilities, project oversight, and invite token generation with tracking.

**Route:** `/admin/manage`

---

## Features

### 1. **Dashboard Overview**
- **Total Users Count** - Display of all registered users in the system
- **Total Projects Count** - Number of active projects
- **Total Cameras** - Combined camera count across all projects

### 2. **User Management**
- **Search & Filter** - Find users by email or display name
- **User List View** - Table with:
  - User avatar (initials in gradient background)
  - Display name and email
  - System role (Admin/User)
  - Number of projects they own/collaborate on
  - Account creation date
  - Delete button with confirmation
  
### 3. **Project Management**
- **Search & Filter** - Find projects by name or owner
- **Project Cards** - Grid layout showing:
  - Project name and owner
  - Total cameras in project
  - Number of collaborators
  - Creation date
  - View button (link to project)
  - Delete button with confirmation

### 4. **Invite Token Generation**
- **One-Click Invite Creation** - Generate single-use invite tokens
- **Auto-Copy Feature** - Click to copy invite URL to clipboard
- **Email Input Validation** - Ensures valid email format
- **Success Feedback** - Toast-style notification with copy confirmation
- **72-Hour Expiry** - Automatic token expiration

### 5. **Invite History**
- **Recent Invites List** - Track all generated invites
- **Time-to-Expiry Counter** - Visual progress bar showing hours remaining
- **Copy-Paste Ready URLs** - Full invite links for easy sharing
- **Expiration Timestamps** - Exact expiry times for planning

### 6. **Safety Features**
- **Delete Confirmation Modal** - Prevents accidental deletions
- **Role-Based Visual Hierarchy** - Admin/User roles clearly indicated
- **Loading States** - User feedback during async operations

### 2. Create API Hooks

Create `packages/frontend/src/api/admin.ts`:

```typescript
import axios from 'axios';
import { useQuery, useMutation } from '@tanstack/react-query';

const API_BASE = '/api/v1';

// Hooks
export const useAdminUsers = () => {
  return useQuery({
    queryKey: ['admin', 'users'],
    queryFn: async () => {
      const res = await axios.get(`${API_BASE}/admin/users`);
      return res.data.users;
    },
  });
};

export const useAdminProjects = () => {
  return useQuery({
    queryKey: ['admin', 'projects'],
    queryFn: async () => {
      const res = await axios.get(`${API_BASE}/projects`);
      return res.data.projects;
    },
  });
};

export const useGenerateInvite = () => {
  return useMutation({
    mutationFn: async (email: string) => {
      const res = await axios.post(`${API_BASE}/admin/invite`, { email });
      return res.data;
    },
  });
};

export const useDeleteUser = () => {
  return useMutation({
    mutationFn: async (userId: string) => {
      await axios.delete(`${API_BASE}/admin/users/${userId}`);
    },
  });
};

export const useDeleteProject = () => {
  return useMutation({
    mutationFn: async (projectId: string) => {
      await axios.delete(`${API_BASE}/projects/${projectId}`);
    },
  });
};
```

### 3. Create Route Handler

Create `packages/frontend/src/pages/AdminPage.tsx`:

```typescript
import React, { useEffect } from 'react';
import { useNavigate } from 'react-router-dom';
import AdminDashboard from '../components/admin/AdminDashboard';
import { useAuth } from '../hooks/useAuth';

export default function AdminPage() {
  const navigate = useNavigate();
  const { user } = useAuth();

  useEffect(() => {
    // Redirect non-admins
    if (user && user.system_role !== 'admin') {
      navigate('/');
    }
  }, [user, navigate]);

  if (!user || user.system_role !== 'admin') {
    return <div>Unauthorized</div>;
  }

  return <AdminDashboard />;
}
```

## Security Considerations

1. **Admin Authorization** - Always verify `user.system_role === 'admin'` server-side
2. **Rate Limiting** - Implement on invite and delete endpoints
3. **Audit Logging** - Log all admin actions (user deletion, invite generation)
4. **CSRF Protection** - Ensure POST requests include CSRF tokens
5. **Input Validation** - Server validates all email addresses and IDs

## Support & Troubleshooting

### Common Issues

**Invite URL doesn't work:**
- Verify backend endpoint is accessible
- Check that token format matches between frontend and backend
- Ensure 72-hour TTL matches between systems

**Users/Projects not loading:**
- Check network tab for API errors
- Verify authorization token is valid
- Ensure user has admin role server-side

**Delete operations fail silently:**
- Add error logging to catch block
- Implement error toast notifications
- Check browser console for stack traces
